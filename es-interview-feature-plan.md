# ES深掘り面接機能 追加仕様案

作成日: 2026-06-25

この資料は、AI面接官アプリに追加する予定の「ESを読み込み、その内容をもとに面接質問・深掘り質問を行う機能」の仕様案。  
まだ実装しない。今後の実装前に参照する機能整理として記録する。

## 参考アプリから読み取ったAI利用方針

参考URL: https://askseed.siders.jp/#/home

公開ページ・公開JavaScriptから確認できた範囲では、参考アプリは以下のような構成に見える。

- Flutter Webアプリとして作られている。
- Google Sign-Inを使う。
- Firebase系のデータ保存・認証を使っている可能性が高い。
- AI処理には `Gemini AI` / `GeminiAIService` / `gemini-2.5-flash` の文字列が含まれる。
- AIアクションとして `followup_questions` と `answer_improvement` が確認できる。
- ES関連のAIフィードバックとして `AiEsFeedback`、`AiEsScores`、`questionIntent`、`readability`、`structure`、`specificity`、`etiquette`、`overallVerdict`、`strengths`、`improvements` が確認できる。
- 面接練習データには `answer`、`answerCount`、`answerTimerSec`、`avgAnswerSec`、`interviewStages`、`lastScore`、`parentAnswerId`、`parentId`、`questionType`、`sourceEsEntryId`、`questionTextSnapshot` などの項目がある。

このアプリでは、参考アプリの「ESをAIで分析し、質問意図・改善点・回答改善・深掘り質問に使う」考え方を取り入れる。

## 追加する目的

通常の面接では、事前に提出したES、履歴書、職務経歴書、応募フォームの内容をもとに質問される。  
そのため、AI面接官アプリにもES入力機能を入れ、AIがES内容を読んだうえで、実際の面接に近い深掘り質問を行えるようにする。

## 基本コンセプト

ユーザーがESを入力する。  
AIがESの内容を分析する。  
AI面接官がESの記述を根拠に質問する。  
ユーザーの回答に対して、さらに深掘りする。  
最後に、ES本文と面接回答の一貫性・具体性・説得力を評価する。

## 想定する利用フロー

1. ユーザーが企業・職種を登録する。
2. ユーザーがES項目を入力する。
3. AIがESを分析し、質問意図・強み・弱点・深掘り候補を抽出する。
4. ユーザーが面接練習を開始する。
5. AI面接官がESの内容をもとに質問する。
6. ユーザーが回答する。
7. AIが回答をES内容と照合し、追加質問を出す。
8. 面接終了後、ESと回答の整合性、具体性、改善点をレポートする。

## 画面として追加する機能

### 1. ES管理画面

- 企業ごとにESを登録できる。
- 職種・部署・応募区分を紐づけられる。
- 複数のES設問を登録できる。
- 設問、回答、文字数制限、提出状況、更新日を保存できる。
- ES項目を一覧で確認できる。
- 面接練習に使うES項目を選択できる。

### 2. ES入力フォーム

- 設問文を入力する。
- 回答本文を入力する。
- 文字数制限を入力する。
- カテゴリーを選択する。
  - 自己PR
  - 志望動機
  - ガクチカ
  - 研究・学業
  - 職務経歴
  - 成功体験
  - 失敗経験
  - 困難克服
  - キャリアプラン
  - その他
- 保存前に文字数を表示する。
- AI分析を実行するボタンを置く。

### 3. ES分析結果画面

- 設問の意図を表示する。
- ES回答の要約を表示する。
- 回答内の主張を抽出する。
- 根拠・数字・成果・学びを抽出する。
- 不足している情報を表示する。
- 深掘りされやすいポイントを表示する。
- 改善案を表示する。
- 面接で聞かれそうな質問を表示する。

### 4. ESベース面接モード

- 面接開始時に、使用するESを選ぶ。
- AI面接官はESの内容を前提に質問する。
- 1問目はES設問に対応した基本質問にする。
- 2問目以降はユーザー回答を踏まえて深掘りする。
- 質問ごとに、どのES項目を根拠にした質問かを内部的に保持する。
- ユーザーには必要に応じて「ESのこの内容について伺います」と表示する。

### 5. ES・面接回答比較レポート

- ES本文と面接回答が一貫しているかを評価する。
- ESに書いた内容を面接で十分説明できたかを評価する。
- 回答がESの焼き直しだけになっていないかを見る。
- ESよりも具体的な補足ができたかを見る。
- ESと矛盾する発言がないかを見る。
- 深掘り質問への耐性を評価する。
- 次回までに補強すべきエピソードを表示する。

## AIで行う処理

### 1. ES設問意図の分類

ES設問が何を見ているかを分類する。

- 志望度
- 企業理解
- 職種理解
- 自己PR
- 強み
- 価値観
- 行動特性
- 課題解決力
- リーダーシップ
- チームワーク
- 学習力
- 再現性
- キャリア観

### 2. ES本文の構造分析

ES本文から以下を抽出する。

- 結論
- 背景
- 課題
- 自分の役割
- 具体的な行動
- 成果
- 数字・事実
- 学び
- 入社後の活用
- 曖昧な表現
- 面接で突っ込まれやすい表現

### 3. スコアリング

ESを以下の観点で評価する。

- 質問意図への回答度
- 読みやすさ
- 構成
- 具体性
- 企業・職種との接続
- 自分の役割の明確さ
- 成果の説得力
- 学び・再現性
- 面接で説明しやすいか
- マナー・表現の適切さ

### 4. 深掘り質問生成

ESから深掘り質問を生成する。

- なぜその行動を選んだのですか。
- その中であなた自身が担当した範囲はどこですか。
- 他に選択肢はありましたか。
- 結果を数字で表すとどうなりますか。
- 失敗や反省点はありましたか。
- 周囲はどのように反応しましたか。
- もう一度同じ状況なら何を変えますか。
- その経験は当社でどう活かせますか。
- ESに書かれていない補足情報はありますか。
- 面接官が確認できる事実や根拠はありますか。

### 5. 回答改善

ユーザーの面接回答に対して、以下を返す。

- 良い点
- 改善点
- ESとの整合性
- 具体性の不足
- 追加すべき数字・事実
- 話す順序の改善
- より良い回答例
- 次の深掘りに備えるポイント

## 面接官の質問生成ルール

- ES本文に書かれている内容を根拠に質問する。
- 一度に複数の質問を投げない。
- 最初は広く聞き、次に具体的に深掘りする。
- ES本文をそのまま読み上げるだけの質問にしない。
- ユーザーの回答が抽象的な場合は、数字・役割・判断理由を聞く。
- ESと回答が矛盾した場合は、責めるのではなく確認する形で聞く。
- 公正採用に反する個人情報は質問しない。
- 本籍、家族、宗教、思想信条、支持政党などは扱わない。
- 回答改善は厳しすぎず、面接で使える具体案に落とす。

## 既存の質問集との連携

[interview-question-bank.md](./interview-question-bank.md) の質問を、ES内容に合わせて使い分ける。

- ESに自己PRがある場合: 自己PR・強み、深掘り質問を優先する。
- ESに志望動機がある場合: 志望動機、企業研究、職種理解を優先する。
- ESにガクチカがある場合: 新卒向け、経験・成果、失敗・困難を優先する。
- ESに職務経歴がある場合: 転職・中途、成果、職種別質問を優先する。
- ESに研究や開発経験がある場合: 技術職、技術面接、思考プロセスを優先する。
- ESの内容が抽象的な場合: 深掘り質問を増やす。
- ESと回答がずれた場合: 整合性確認の質問を出す。

## MVPで入れる範囲

最初の実装では、以下まででよい。

1. ESを手入力で登録する。
2. ES設問と回答を保存する。
3. AIがESを要約し、質問意図・改善点・深掘り候補を出す。
4. ESを1つ選んで面接練習を開始する。
5. AI面接官がESに基づいて質問する。
6. 面接終了後、ESと回答の整合性をフィードバックする。

## 注意点

- ESには個人情報が含まれる可能性があるため、保存・AI送信の説明を明確にする。
- AIに送る内容は必要最小限にする。
- ユーザーが削除したESは、関連する分析結果も削除できるようにする。
- AIの評価は採用判断ではなく練習用フィードバックとして表示する。
- 公正採用に反する質問をAIが生成しないようにガードする。
- ES本文を丸暗記させるのではなく、面接で自然に説明できる状態を目標にする。
