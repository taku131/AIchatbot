AIchatbot Project Plan

AI面接官で面接対策をするアプリ

AI面接官が質問し、ユーザーの回答に深掘りし、最後に改善点と回答例を返す面接練習アプリを作成する。 まずはMVPとして「モード選択」「面接チャット」「フィードバック」までを実装する。

開発ゴール

ブラウザ上で動くAI面接練習アプリを作る。 ユーザーは面接モードを選び、AI面接官と会話し、面接終了後に評価と改善案を確認できる。

AI面接官 面接練習 チャットUI フィードバック OpenAI API

推奨技術

  • フロントエンド: React + Vite
  • UI: Tailwind CSS
  • バックエンド: Node.js + Express
  • AI連携: OpenAI API
  • 保存: 最初はローカル保存、後でSQLite

主要機能

MVP

面接モード選択

新卒、転職、技術面接、英語面接、自己PR対策などから練習内容を選べる。

MVP

AI面接官チャット

AIが質問し、ユーザー回答に応じて深掘り質問を返す。面接らしい流れを作る。

MVP

フィードバック

回答の良い点、改善点、より良い回答例、次回の練習ポイントを表示する。

次段階

評価スコア

論理性、具体性、志望度、コミュニケーション力などをスコア化する。

次段階

履歴保存

過去の面接内容、回答、フィードバックを保存し、後から見返せるようにする。

拡張

音声入力

マイクで回答し、実際の面接に近い練習をできるようにする。

実装ロードマップ

Phase 1

基本アプリ作成

React + Viteで画面を作り、モード選択、チャット画面、結果画面の見た目を完成させる。

Phase 2

AI連携

Expressバックエンドを作成し、OpenAI APIに質問生成とフィードバック生成を依頼する。

Phase 3

面接品質の調整

面接官プロンプトを改善し、深掘り質問、評価基準、回答例の品質を上げる。

Phase 4

保存と拡張

履歴保存、評価スコア、音声入力、職種別テンプレートを追加する。

MVPで作る範囲

優先度 機能 内容
高 面接モード選択 練習したい面接タイプを選ぶ。
高 AI面接官チャット 質問、回答、深掘り質問をチャット形式で行う。
高 面接終了 任意のタイミングで面接を終了し、評価生成に進む。
高 フィードバック表示 良い点、改善点、回答例、次回の練習ポイントを出す。

最初に読むべき事項: 面接対策の全体像

3つの調査エージェントの回答を統合し、AI面接官アプリで最初に読ませるべき面接対策事項として整理したもの。 暗記用ではなく、面接で評価される観点を理解し、練習時のチェック項目に変換するための基礎資料。

読む順序

面接の目的と評価観点

面接は回答暗記の発表ではなく、職務適合性、能力、価値観、志望度、再現性、対話力を確認する場。

自己分析と経験の棚卸し

強み、弱み、価値観、仕事選びの軸、成功、失敗、困難、協働、リーダーシップ、学習経験を整理する。

企業研究・職種研究

事業、顧客、競合、職務内容、必要スキル、成果指標、カルチャー、求める人物像を調べる。

回答の基本構造

結論、理由、具体例、成果、学び、入社後の活用の順で話す。行動面接ではSTAR法を基本型にする。

深掘り質問への備え

なぜそうしたか、他案は何か、自分の役割は何か、失敗要因は何か、再現できるかを説明できるようにする。

当日の振る舞いとオンライン準備

時間、服装、姿勢、表情、声、聞く姿勢、通信、カメラ、照明、背景、通知オフを整える。

逆質問・面接後フォロー

役割、成果基準、チーム、課題、入社後の期待を聞く。面接後は簡潔な感謝と関心を伝える。

対策すべき重要事項

面接の目的理解

面接官は話の上手さだけでなく、職務に必要な知識、スキル、経験、価値観、問題解決力、組織との相性を見ている。

質問意図の理解

自己PR、志望動機、強み弱み、失敗経験には、それぞれ確認したい評価観点がある。質問の裏にある意図を読んで答える。

自己分析

過去経験から、自分の強み、弱み、価値観、やりがい、苦手な環境、仕事選びの軸を言語化する。

経験の棚卸し

学業、仕事、研究、アルバイト、部活、プロジェクト、失敗、挫折、協働、改善、学習の経験をリスト化する。

強みの根拠

「私は粘り強いです」だけで終えず、その強みが表れた状況、行動、成果、周囲への影響を添える。

弱みの扱い

致命的な印象や責任転嫁を避け、弱みの認識、改善行動、現在の変化まで話す。

企業研究

企業理念、事業内容、顧客、収益源、競合、強み弱み、採用情報、選考フローを確認する。

職種研究

仕事内容、必要スキル、成果指標、関係部署、入社後の期待役割を理解し、自分の経験と接続する。

志望動機

なぜ業界、なぜ企業、なぜ職種、入社後に何をしたいかを、自分の経験と企業特徴の接点で説明する。

応募書類との整合

履歴書、ES、職務経歴書に書いた内容を自然に説明できるようにし、面接回答と矛盾させない。

基本質問対策

自己紹介、自己PR、志望動機、ガクチカ、実績、転職理由、強み弱み、失敗経験、困難克服、将来像、逆質問を準備する。

回答構成

結論を先に置き、根拠、具体例、結果、学び、次にどう活かすかの順で短く話す。

STAR法

Situation、Task、Action、Resultで経験を整理する。特に「自分が何を考え、何をしたか」を明確にする。

STARにLearningを足す

失敗、困難、改善系の質問では、結果で終わらず、学びと次回への応用まで話す。

具体性

人数、期間、売上、工数、品質、速度、満足度、再発防止など、数字や事実で成果を示す。

自分の役割

チーム成果を話す場合は、チーム全体の話と自分の担当、判断、行動、影響を分ける。

深掘り耐性

なぜ、具体的には、他の選択肢は、反対意見は、失敗要因は、再現性は、という追加質問に答えられるようにする。

行動面接

過去の行動から将来の行動を推定される。リーダーシップ、対立、失敗、改善、巻き込み、学習、倫理観の実例を用意する。

状況面接

「もしこうなったら」系では、前提確認、優先順位、リスク、関係者、判断基準を言語化する。

技術面接

答えだけでなく、要件確認、制約整理、設計方針、実装、テスト、計算量、例外処理、トレードオフ説明が見られる。

職種別評価軸

営業は顧客理解と数字、PMは課題設定と合意形成、エンジニアは設計と保守性、マネージャーは育成と意思決定を重視する。

転職理由・退職理由

不満の羅列にせず、学び、今後実現したいこと、応募先との接点に変換する。

答えにくい質問

ブランク、短期離職、未経験、希望年収、失敗経験は、誠実さと改善行動を軸に説明を準備する。

面接フェーズ別対策

一次は基本適性、二次は職務理解と再現性、最終は志望度、価値観、入社意思が見られやすい。

話す量と時間

1回答は目安1から2分。長い前置きを避け、深掘りされたら追加説明する。

話し方

声量、速度、間、語尾、明瞭さを整える。フィラーを減らし、オンラインでは少しはっきり話す。

聞く姿勢

うなずき、メモ、相手の発言を踏まえた返答で、対話できることを示す。

第一印象

服装、表情、姿勢、挨拶、時間厳守、受付や待機中の態度は内容以前の基本評価に影響する。

オンライン面接

通信、カメラ、マイク、照明、背景、通知オフ、充電、予備イヤホン、名前表示を事前確認する。

台本読み回避

メモはキーワード程度にし、画面読み上げではなく自然な会話として話す。

逆質問

役割、成果基準、チーム、事業課題、入社後の期待、選考プロセスを聞く。待遇だけに偏らない。

NG行動

遅刻、準備不足、スマホ確認、相手を遮る、話しすぎ、質問なし、カメラオフ、前職批判、虚偽、過剰な自己PRを避ける。

公正採用への配慮

本籍、家族、宗教、支持政党、思想信条、生活環境など、適性や能力に関係ない事項を質問・評価に使わない。

面接後フォロー

面接後は、感謝、印象に残った話、職務への関心、自分の適合点を簡潔に伝える。

メンタル準備

睡眠、軽い運動、深呼吸、経路確認、想定質問練習で不安要因を減らす。緊張は普通の反応として扱う。

AI面接官アプリに入れる評価チェック項目

  • 面接の目的と評価観点を理解しているか
  • 自己分析の強み、弱み、価値観、仕事選びの軸が入力済みか
  • 成功、失敗、困難、協働、リーダーシップ、学習のエピソードがあるか
  • 企業の事業、顧客、競合、職種、求める人物像を調べているか
  • 志望動機に「なぜ業界」「なぜ企業」「なぜ職種」があるか
  • 応募書類と面接回答に矛盾がないか
  • 回答が結論から始まっているか
  • STARの状況、課題、行動、結果が揃っているか
  • 自分の行動とチーム成果を分けて説明しているか
  • 成果が数字、期間、対象、変化で示されているか
  • 学び、改善、次回への応用があるか
  • 質問に対して過不足なく答えているか
  • 回答が長すぎないか、短すぎないか
  • 抽象語だけで終わっていないか
  • 深掘り質問に耐えられる具体性があるか
  • 失敗談で責任転嫁していないか
  • 退職理由や弱みを前向きな改善につなげているか
  • 技術面接で前提確認、設計、テスト、計算量、トレードオフを話せているか
  • 職種別の評価軸に回答が合っているか
  • 声量、速度、間、語尾、フィラーに問題がないか
  • 表情、姿勢、視線、うなずき、聞く姿勢が整っているか
  • オンライン環境のカメラ、マイク、照明、背景、通知オフを確認したか
  • 逆質問が3つ以上あり、待遇だけに偏っていないか
  • 前職批判、虚偽、話しすぎ、質問なし、台本読みを検知できるか
  • 本籍、家族、宗教、思想信条など職務無関係な属性を質問・評価していないか
  • 面接後の振り返りとして、良かった点、改善点、次回練習テーマを出せるか

統合に使った参考観点

3名のエージェントが参照・整理した観点を統合した。国内の就活・転職面接対策、構造化面接、STAR法、 行動面接、技術面接、オンライン面接、面接後フォロー、公正採用の観点を含める。
